1. Difference between Link and NavLink
Link
Used to navigate from one page to another.
It does not know which page is currently active.

Example:

<Link to="/about">About</Link>
NavLink
Also used for navigation.
It knows if the current route is active.
Useful for navigation bars because you can highlight the active link.

Example:

<NavLink to="/about">About</NavLink>
🌍 Real-life Example

On YouTube, the current page (Home, Shorts, Subscriptions) is highlighted.

That's similar to how NavLink works.

2. What is BrowserRouter?

Answer

"BrowserRouter enables routing in a React application. It keeps the UI and browser URL in sync using the browser's History API."

Without BrowserRouter, routing won't work.

Example:

<BrowserRouter>
   <App />
</BrowserRouter>
🌍 Real-life Example

Think of it as the manager of all routes. It watches the URL and tells React which page to show.

3. What is Routes?

Answer

"Routes contains all the route definitions. It checks the current URL and renders the matching route."

Example:

<Routes>
   <Route path="/" element={<Home />} />
   <Route path="/about" element={<About />} />
</Routes>
🌍 Real-life Example

Think of Routes as a security guard checking your ticket (URL) and sending you to the correct room (component).

4. What is Route?

Answer

"Route maps a URL path to a React component."

Example:

<Route
    path="/about"
    element={<About />}
/>

Meaning:

If the URL is

/about

show

<About />
5. Why not use <a>?
❌
<a href="/about">About</a>

Problem:

Reloads the whole page.
Loses component state.
Slower.
✅
<Link to="/about">

No page refresh.

Faster.

Better user experience.

🌍 Example

Imagine Netflix.

Clicking Movies doesn't reload the whole website.

That's because it uses client-side routing.

6. What is Client-side Routing?

Answer

"Client-side routing changes the displayed page without reloading the browser."

Only the content changes.

The whole page doesn't reload.

Answer

10. What is the History API?

Answer

"The History API lets JavaScript change the browser URL without refreshing the page."

React Router uses this API internally.
